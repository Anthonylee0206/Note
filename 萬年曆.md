# SQL：製作萬年曆 (Calendar)

----

這份筆記紀錄了如何利用 SQL 迴圈與日期函數，從零建立一個連續日期的資料表，最後運用 `PIVOT` 語法將資料轉換成實體的月曆格式。

---

## 階段一：基礎版 - 建立連續日期清單與假日判斷

前輩展示了兩種寫法，邏輯是一樣的，差別只在於「要不要先把值塞給變數」。
這個階段的核心是學會使用 `WHILE` 迴圈搭配 `DATEADD()` 讓日期一天一天往前推進。

### 寫法 A：直接把函數寫在 INSERT 裡面 (乾淨俐落)
這段語法會跑出一整年 (2026) 的日期清單，並標註假日。

```sql
-- 1. 先把舊的表刪掉 (如果存在的話)，避免重複執行報錯
DROP TABLE IF EXISTS Calendar;

-- 2. 建立存放日期的資料表
CREATE TABLE Calendar
(
    [DATE] DATE,        -- 完整日期 (例：2026-01-01)
    [年] INT,
    [月] INT,
    [日] INT,
    [禮拜幾] INT,       -- 存數字 (通常預設 1是星期日、7是星期六)
    [假日] VARCHAR(1)   -- 用 'v' 來標記假日
);

-- 3. 設定迴圈的起始與結束條件
DECLARE @BeginDate DATE = '2026-01-01';
DECLARE @EndDate DATE = '2026-12-31';
DECLARE @CurrentDate DATE = @BeginDate; -- 將當前日期先設為第一天

-- 4. 開始跑迴圈：只要當前日期小於等於結束日期，就繼續跑
WHILE (@CurrentDate <= @EndDate)
BEGIN
    -- 將算好的資料塞進資料表
	INSERT INTO Calendar([DATE], [年], [月], [日], [禮拜幾], [假日])
	VALUES
	(
		@CurrentDate,
		YEAR(@CurrentDate),                 -- 擷取年份
		MONTH(@CurrentDate),                -- 擷取月份
		DAY(@CurrentDate),                  -- 擷取日期
		DATEPART(WEEKDAY, @CurrentDate),    -- 算出這天是星期幾 (回傳數字)
        
        -- 【注意💡】這裡前輩用 IIF 判斷，如果星期幾是 6 或 7 就填 'v'。
        -- 註：如果系統預設星期日是 1，那 6和7 代表星期五和星期六。
        -- 如果要標註「六日」為假日，可能要改成 IN (1, 7) 哦！
		IIF(DATEPART(WEEKDAY, @CurrentDate) IN (6, 7), 'v', '')
	);
    
    -- 5. 【關鍵步】把當前日期加 1 天，這樣迴圈才會往前走，不然會變死迴圈！
	SET @CurrentDate = DATEADD(DAY, 1, @CurrentDate);
END

-- 6. 查看結果
SELECT * FROM Calendar;
```
### 寫法 B (變數指派法)
把 YEAR()、MONTH() 這些函數先算好，存進 @Y, @M 變數裡，再 INSERT 進去。對於初學者來說，這種「一步一步算好再放進去」的寫法在除錯 (Debug) 時會比較好懂。

## 階段二：進階版 - 用 PIVOT 畫出真實月曆格式
前面我們產生的資料是「直直往下長」的（一天一列）。但真正的月曆是橫向發展的（星期日到星期六）。這裡必須用到 SQL 的大絕招：樞紐分析 PIVOT。
```sql
-- 1. 設定系統的「每個禮拜第一天」是星期日 (7 = Sunday)
-- 這步很重要，統一標準才不會因為不同電腦設定導致排版歪掉
SET DATEFIRST 7; 

DROP TABLE IF EXISTS Calendar;
CREATE TABLE Calendar (
    [Date] DATE,
    [Year] INT,
    [Month] INT,
    [Day] INT,
    [Weekday] INT,      -- 星期幾 (1~7)
    [WeekNumber] INT    -- 【關鍵】這一天是今年的第幾個禮拜 (用來做換行的依據)
);

-- 2. 一樣跑迴圈塞資料 (這裡省略假日判斷，專注在排版需要的欄位)
DECLARE @CurrentDate DATE = '2026-01-01';
DECLARE @EndDate DATE = '2026-12-31';
WHILE (@CurrentDate <= @EndDate)
BEGIN
    INSERT INTO Calendar ([Date], [Year], [Month], [Day], [Weekday], [WeekNumber])
    VALUES (
        @CurrentDate, 
        YEAR(@CurrentDate),
        MONTH(@CurrentDate),
        DAY(@CurrentDate),
        DATEPART(WEEKDAY, @CurrentDate), 
        DATEPART(WEEK, @CurrentDate)  -- 取出這天是全年的第幾週
    );
    SET @CurrentDate = DATEADD(DAY, 1, @CurrentDate);
END

-- 3. 施展 PIVOT 魔法，把列轉成欄位
SELECT 
    [Year],
    [Month],
    -- 下面這些 [1] 到 [7] 就是 PIVOT 轉出來的新欄位 (對應星期日到六)
    MAX([1]) AS [Sun],    
    MAX([2]) AS [Mon],
    MAX([3]) AS [Tue],
    MAX([4]) AS [Wed],
    MAX([5]) AS [Thu],
    MAX([6]) AS [Fri],
    MAX([7]) AS [Sat]
FROM Calendar
PIVOT
(
    -- PIVOT 語法：把什麼東西當作裡面的值？ -> 把 [Day] (日期) 放進去
    -- 因為 PIVOT 規定要搭配聚合函數，所以這裡隨便套一個 MAX 就好
    MAX([DAY]) 
    
    -- 把哪一欄的值拿來當作新的欄位名稱？ -> 把 [Weekday] 裡面的 1~7 轉成欄位
    FOR [Weekday] IN ([1],[2],[3],[4],[5],[6],[7])
) AS pvt

-- 把同一年、同一月、同一週的資料群組在同一列 (這就是為什麼前面要抓 WeekNumber)
GROUP BY
    [Year], 
    [Month], 
    [WeekNumber]
    
-- 照順序排好，月曆才不會亂跳
ORDER BY 
    [Year], 
    [Month], 
    [WeekNumber]
```
## 核心必背語法
- DATEADD(DAY, 1, 日期)：用來將日期加一天。

- DATEPART(WEEKDAY, 日期)：取得該日期是星期幾（回傳數字）。

- DATEPART(WEEK, 日期)：取得該日期是當年的第幾週。

- IIF(條件, 成立回傳這個, 不成立回傳這個)：好用的單行判斷式。

- PIVOT：將列資料轉為欄位資料的強大工具。
