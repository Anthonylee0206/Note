# 全部 SP 共同問題彙總

分析六支 SP 後發現以下**團隊層面的共同問題**，建議跟主管提出全面清查：

| 共同問題 | 出現次數 | 出現在哪些 SP |
|---|---|---|
| `SET NOCOUNT, ARITHABORT OFF` 不需要 | 6 次 | 全部 |
| MERGE 缺 HOLDLOCK | 4 次 | SP 2, 5, 6 |
| CATCH 缺 `@@TRANCOUNT` 和 `THROW` | 3 次 | SP 2, 5, 6 |
| ISNULL 包 WHERE 欄位（sargability） | 3 次 | SP 2, 4, 6 |
| `dbo.GetCurrArray` SQL Injection | 2 次 | SP 3, 4 |
| FORMAT() 效能問題 | 2 次 | SP 2, 5 |
| BETWEEN 日期邊界 | 2 次 | SP 3, 4 |
| 同樣查詢跑兩次（資料 + COUNT） | 2 次 | SP 3, 4 |
| `mem_promotion.idx_is_active` 缺 INCLUDE | 2 次 | SP 1, 5 |
| 舊式 DROP TABLE | 3 次 | SP 2, 5, 6 |

### 建議團隊行動

1. **統一 coding standard**：把 SET OFF、HOLDLOCK、CATCH 模式寫進團隊規範
2. **全面清查 `dbo.GetCurrArray`**：所有用到這個函數的 SP 都有 SQL Injection 風險
3. **全面清查 FORMAT()**：所有用到的地方都改成 CONVERT 或 DATEADD
4. **索引整合**：`mem_promotion.idx_is_active` 一次加好涵蓋多支 SP 的 INCLUDE
